# Resume System

This directory contains the **single source of truth** for the resume.
Two PDF variants are automatically generated by GitHub Actions and committed to `Assets/`.

| File | Purpose |
|------|---------|
| `resume.md` | **Edit this.** All content lives here. |
| `templates/plain.html` | ATS-first HTML template (minimal styling) |
| `templates/designed.html` | Visually designed HTML template |
| `styles/plain.css` | Styles for the plain/ATS PDF |
| `styles/designed.css` | Styles for the designed PDF |

---

## How to edit the resume

1. Open `resume/resume.md` in any text editor (or ask an AI assistant to update it).
2. Follow the `<!-- TODO: -->` markers for every section that needs your real content.
3. Commit and push to `main` — GitHub Actions will automatically rebuild both PDFs.

### Useful Markdown tips

```markdown
# Your Name                ← h1 = your full name
**Label:** value | ...     ← contact line (pipe-separated)

## Section Heading         ← h2 = major section
### Role — Company         ← h3 = job / sub-entry

- Bullet point             ← accomplishment bullets
```

---

## Generated PDFs

| Path | Description |
|------|-------------|
| `Assets/Resume-ATS.pdf` | Plain, ATS-optimised (no columns, clean text) |
| `Assets/Resume-Designed.pdf` | Visually designed (two-column sidebar layout) |

Both files are committed automatically.  
Your GitHub Pages site can link to them at:

```
/Assets/Resume-ATS.pdf
/Assets/Resume-Designed.pdf
```

---

## Running the build locally

```bash
# Install dependencies (once)
npm install
npx playwright install chromium --with-deps

# Build both PDFs
npm run build:resume
```

The PDFs are written to `Assets/Resume-ATS.pdf` and `Assets/Resume-Designed.pdf`.

---

## Workflow file

The GitHub Actions workflow is at `.github/workflows/build-resume.yml`.  
It runs on every push to `main` (excluding commits that only modify the PDFs) and can also be triggered manually via **Actions → Build Resume PDFs → Run workflow**.
